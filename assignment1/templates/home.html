About Stack Overflow Activity Tracker:
Chart to visualize user behavior on Stack Overflow:

1. linksvisited:

linkvisited event is recognized as those events that a user navigates from one hyperlink to another. Count of links visited by a user helps in understanding the breadth of user's interest in using StackOverflow for Java.
Intention of a user behind using Stack Overflow can be drawn by knowing the number of links a user visits.
For example: If the user frequently visits multiple Java-related questions, it could mean that the user is either new to Java or is currently working on a Java project that needs constant learning.
On the other hand, if the user has visited Stack Overflow only once in a long time, it could mean that the user has prior Java knowledge or user is not working on a Java project.

2. questionsandanswers:

questions and answers event are recognized as an event when a user posts an answer to a question or a question who's answer he is looking for. Important part of StackOverflow community is to actively share and gain knowledge by asking questions and posting answers that a user is familiar with.
This component has more weightage as a user is not only contributing to expansion of Stack Overflow Knowledge Base as well as his skill in Java. His/her interest to help and encourage other Java developers.

3. upvotecount:

upvote is an event that a user marks on the website to indicate favorability and usefulness of the answer. This behavior is tracked to analyze to what extent does a user finds Stack Overflow's Java-based questions and answers useful. How helpful did a user think StackOverflow is.

4. total time logged:

The amount of time a user is logged on in StackOverflow can say a lot about a user, especially with other behaviors tracked above. A user who spends less time on Stack Overflow, has high number of links visited, questions, answers and votes may mean that this user visits stackoverflow regulary, is proud to be part of the community and finds the application very useful. On the other hand, a user may log days on stackoverflow without any activity, this could mean that a user has left Stack Overflow unattended and may not find Stack Overflow engaging enough.

5. text highlights:

Some users that I have noticed highlight text while reading content on StackOverflow. Some Java questions may be hard to understand, user invests certain time understanding some answers by highlighting text. Another use case for this tracked behavior is a user may select text to copy and paste into his personal notes for later reference or finds it important information to highlight. The selected text can also be code snippets that could be used in projects and assignments.

The chart represents total activity that a user finds useful through various behaviors. The chart gives an idea of percentage of activities the user has performed over time. Primary involvement of the user can be drawn from the chart at any point in time.

Insights that can be drawn from above interactive chart for a user:


1. The chart's major trend lies in the time dropdown. By selecting different times from the dropdown, a user can understand which period in the day he/she is most productive. Productivity to a user could mean browsing through StackOverflow's Java related questions. If a user is highly active during the night, this user may be a night person when it comes to learning Java or the person is interested in learning Java as a non-academic skill and hence invests time studying Java during the night time. If a user is highly active during the day, this user may be using Java in an academic project or a work project. The user finds StackOverflow very helpful in project development, fixing bugs, learning new concepts, etc. This helps in understanding overall intention of user using StackOverflow based on time.

2. Apart from viewing the amount of total activity during certain time of the day, a user can also understand which activities are performed most during what time of the day. This is helpful in the sense that, a user may be busy through the day while using StackOverflow's information, but at a later point, the user is available to contribute through answers and upvotes rather than just exploring various links in StackOverflow's knowledge base. This trend can be visualized by looking at the chart during different times in a day from morning to night and back to morning.

3. A user maybe gaining more kowledge during a certain time of the day when he is browsing through various links and upvoting answers and questions that he found useful. A user maybe sharing more knowledge at a different instance of the day, this behavioral trend means that a user is not learning new information at this time, but is contributing with the skill he/she already has about a Java topic. By selecting all 24 hours in a day, a user's behavior can be visualized to interpret the way in which a user has involved himself in StackOverflow.
